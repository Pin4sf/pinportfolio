# My Stack in 2026: What I Use and Why

Every developer has opinions about their tools. Here are mine — not because they're the "best," but because they work for the way I think and build.

## Frontend

**Next.js + TypeScript** is my default. The App Router, server components, and static generation give me the flexibility to build anything from a marketing site to a full-stack app. TypeScript isn't optional — it's the difference between "it works" and "it works reliably."

**GSAP** for animation. CSS animations are great until you need scroll-linked effects, complex timelines, or physics-based motion. GSAP handles all of that with a clean API and consistent 60fps.

**Three.js** when I need 3D. Not every project needs it, but when it does, nothing else comes close. Custom GLSL shaders give you pixel-level control over every visual.

## Backend

**Node.js + Express** for APIs. Fast to prototype, easy to deploy, and JavaScript everywhere means fewer context switches. MongoDB for document storage, Redis for caching and real-time pub/sub.

**Python** for data work. Pandas, NumPy, and Scikit-learn for analysis and ML. TensorFlow when I need deep learning.

## Tools

**Git** is non-negotiable. **Docker** for consistent environments. **Figma** for design. **VS Code** with Vim keybindings because old habits die hard.

## The Philosophy

I don't chase trends. I choose tools that are stable, well-documented, and have strong communities. The best stack is the one that gets out of your way and lets you focus on the problem you're actually solving.
